Provider:- Provider is tool which communicate terraform with diffrent cloud.
(Provider is a plugin in Terraform that enables interaction with a specific cloud or service by defining authentication, region, and API details.)

Block :- Terraform block is the fundamental structure in Terraform configuration that defines how infrastructure should be created and managed.
Type of Block:-

1. Provider Block:- This block tell terraform that which cloud we are working with
2. Resource Block:- This block tell terraform that which or what thing need to create.
3. Variable Block:- A variable block in Terraform is used to define input values that allow Terraform configurations to be flexible, reusable, and environment-independent.
4. Output Block:- This block tell the output of terrafrm code.
5.Module Block:- This block tell terraform to reused which code.
6. Terraform Block:- This block is for terraform setting.
7. Data Block:- It is used to read the information of already existing resources without creating new resource.
8.Dynamic Block:- A dynamic block in Terraform is used to generate repeated nested blocks programmatically using loops.
9. Imports Block:- An import block in Terraform is used to bring existing infrastructure resources under Terraform management by importing them into the Terraform state.
